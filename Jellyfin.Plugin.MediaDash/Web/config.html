MediaDash — Settings

Configure the paths and service names for your setup. All fields are optional — leave blank to disable the associated feature.

Paths

Root directory of your media storage, used for disk usage stats. Example: /mnt/media
Full path to the re-encoder log file. Example: /var/log/reencode_bdremux.log
Full path to the track-stripping log file. Example: /var/log/strip_tracks.log
Full path to the duplicate-detection JSON report. Example: /var/log/find_dupes_report.json
Directory containing per-worker slot JSON files. Example: /tmp/encode_status
JSON file tracking which files have been processed. Example: /var/lib/reencode/processed.json
Touch this file to pause encoding; delete it to resume. Example: /tmp/reencode_pause
Touch this file to force-resume past quiet hours. Example: /tmp/reencode_force
Full path to the find_dupes.py script. Example: /root/scripts/find_dupes.py

Service Names

Service name passed to systemctl start. Example: reencode-bdremux
Service name passed to systemctl start. Example: strip-tracks

Process Detection

Substring to match in /proc/*/cmdline to detect an active encode. Example: reencode_bdremux
Substring to match in /proc/*/cmdline to detect active stripping. Example: strip_tracks

Quiet Hours

Hour when encoding should pause. This is stored in the plugin config and read by your scripts via the status API.
Hour when encoding should resume.

Queue Scanner

Comma-separated list of file extensions to scan for the encode queue. Example: .mkv,.mp4,.avi,.m2ts
Comma-separated codec names that are already done (not counted in queue). Example: hevc,av1
